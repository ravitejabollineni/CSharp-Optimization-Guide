# Missing Number Problem

## Problem Statement

Given an array `nums` containing `n` distinct numbers in the range `[0, n]`, return the only number in the range that is missing from the array.

### Examples

**Example 1:**
```
Input: nums = [3,0,1]
Output: 2
```
**Explanation:** `n = 3` since there are 3 numbers, so all numbers are in the range `[0,3]`. 2 is the missing number in the range since it does not appear in `nums`.

**Example 2:**
```
Input: nums = [0,1]
Output: 2
```
**Explanation:** `n = 2` since there are 2 numbers, so all numbers are in the range `[0,2]`. 2 is the missing number in the range since it does not appear in `nums`.

**Example 3:**
```
Input: nums = [9,6,4,2,3,5,7,0,1]
Output: 8
```
**Explanation:** `n = 9` since there are 9 numbers, so all numbers are in the range `[0,9]`. 8 is the missing number in the range since it does not appear in `nums`.

### Constraints

- `n == nums.length`
- `1 <= n <= 10^4`
- `0 <= nums[i] <= n`
- All the numbers of `nums` are unique

---

## Solution Approach

### Key Insight: Mathematical Sum Formula

The most efficient solution uses the **Gauss Sum Formula** to find the missing number. The sum of numbers from 0 to n is:

```
Sum = n × (n + 1) / 2
```

By calculating the expected sum and subtracting the actual sum of the array, we get the missing number.

### Algorithm Steps

1. Calculate the expected sum of numbers from 0 to n using the formula
2. Calculate the actual sum of all elements in the array
3. The difference is the missing number

---

## Solution Implementation

### Solution 1: Compact Version

```csharp
public class Solution {
    public int MissingNumber(int[] nums) {
        int sum = nums.Length * (nums.Length + 1) / 2;
        return sum - nums.Sum();
    }
}
```

### Solution 2: Explicit Version

```csharp
public static int FindMissingNumber(int[] nums) {
    int n = nums.Length;
    int expectedSum = n * (n + 1) / 2;  // Sum of 0 to n
    int actualSum = 0;
    
    foreach (int num in nums) {
        actualSum += num;
    }
    
    return expectedSum - actualSum;
}
```

---

## Explanation

### Why This Works

1. **Expected Sum Calculation:**
   - If we have an array of length `n`, the complete range is `[0, n]`
   - The sum of all numbers from 0 to n = `n × (n + 1) / 2`
   - Example: For n=3, range is [0,1,2,3], sum = 3×4/2 = 6

2. **Actual Sum Calculation:**
   - We sum all the numbers present in the array
   - Example: For [3,0,1], actual sum = 3+0+1 = 4

3. **Finding the Missing Number:**
   - Missing number = Expected sum - Actual sum
   - Example: 6 - 4 = 2 ✓

### Time & Space Complexity

- **Time Complexity:** O(n) - We iterate through the array once
- **Space Complexity:** O(1) - We only use a few variables

### Comparison of Solutions

**Solution 1 (Compact):**
- ✅ More concise
- ✅ Uses LINQ's `Sum()` method
- ⚠️ Slightly less efficient (LINQ overhead)

**Solution 2 (Explicit):**
- ✅ More explicit and readable
- ✅ Slightly more efficient (no LINQ overhead)
- ✅ Better for understanding the algorithm

## Alternative Approaches

### XOR Approach (Also O(n) time, O(1) space)

```csharp
public int MissingNumber(int[] nums) {
    int xor = nums.Length;
    for (int i = 0; i < nums.Length; i++) {
        xor ^= i ^ nums[i];
    }
    return xor;
}
```

**How it works:** XOR has the property that `a ^ a = 0` and `a ^ 0 = a`. By XORing all indices and all values, pairs cancel out, leaving only the missing number.
